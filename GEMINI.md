# 프로젝트: 개인 기술 블로그 및 위키

## 1. 프로젝트 목표

- 개인적으로 학습한 IT 기술 지식과 생각을 기록하고 공유하는 웹사이트를 구축한다.
- 블로그의 가독성과 위키의 체계성을 결합하여, 콘텐츠를 효과적으로 전달하고 관리한다.

## 2. 주요 기능 요구사항

- **웹 기반 콘텐츠 관리**: 별도의 프로그램 설치 없이, 웹 UI를 통해 직접 글을 작성, 수정, 삭제할 수 있어야 한다.
- **콘텐츠 유형**:
    - **기술 블로그**: 코드 블록, 인용문 등이 포함된 기술 관련 포스트.
    - **위키**: 체계적으로 정리된 지식 베이스.
- **Git 기반 저장**: 모든 콘텐츠는 Markdown 파일 형태로 GitHub 저장소에 저장되어 버전 관리가 되어야 한다.
- **검색 기능**: 원하는 콘텐츠를 쉽게 찾을 수 있는 검색 기능이 필요하다.

## 3. 디자인 컨셉

- **전체적인 스타일**: 글 내용에 집중할 수 있는 깔끔하고 미니멀한 디자인을 지향한다.
- **시각적 요소**:
    - 다이어그램, 순서도 등의 시각 자료를 적극적으로 활용한다.
    - 시각 자료의 스타일은 **[Excalidraw](https://excalidraw.com/)** 와 같이, 색연필로 그린 듯한 깔끔하고 부드러운 손그림 느낌을 선호한다.

## 4. 핵심 제약사항

- **비용 효율성**: 서버 유지비용이 발생하지 않도록, 가급적 무료 서비스를 최대한 활용하여 구축 및 운영한다.

## 5. 채택된 아키텍처

- **프론트엔드**: React 기반의 정적(Static) 웹사이트
- **백엔드**: Node.js 기반의 서버리스(Serverless) API
- **호스팅**: **Vercel**. 프론트엔드 호스팅과 백엔드 API 실행을 모두 담당한다.
- **데이터 저장소**: GitHub Repository. 콘텐츠(Markdown 파일)와 소스 코드를 함께 관리한다.
- **핵심 워크플로우**:
    1. 사용자가 웹 UI에서 글을 작성하고 저장한다.
    2. 프론트엔드는 백엔드 API로 콘텐츠를 전송한다.
    3. 백엔드 API는 받은 내용을 Markdown 파일로 변환하여 GitHub 저장소에 Commit & Push 한다.
    4. Vercel은 GitHub 저장소의 변경을 감지하여 자동으로 프로젝트를 다시 빌드하고 웹사이트에 배포한다.
